PSI
Processed Source Instructions
What is PSI?
PSI is a language for defining a program's behavior. It is designed to be both human and machine readable, while being able to translate seamlessly into other code, such as assembly or low-level op-codes.
Why use PSI?
PSI lets you describe what a program does without depending on a specific machine or operating system. Its syntax is simple and consistent, making it easy for humans to read while remaining precise for compilers and translators. With PSI, every instruction, type, and constant is clear, ensuring programs behave predictably across different backends. 

// example program

module program;
entry main; // what label program starts from

// check if backend supports following types/metas
required #print;
required $int;
required $str;

constant $str @message "Hello, World!";

// tell backend that sum is a function
function $int sum $int $int;

// definition of sum
sum {
    add %result !0 !1;
    ret %result;
}

// definition of main
main {
    mov $int %x 7;
    mov $int %y 5;
    call sum %result %x %y;
    sys #print @message; 
}
